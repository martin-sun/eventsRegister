# 03 - Internationalization (i18n) / 国际化方案

## Solution: Paraglide JS + SvelteKit

We use [Paraglide JS](https://inlang.com/m/gerre34r/library-inlang-paraglideJs) — purpose-built for SvelteKit. It generates **type-safe translation functions at compile time** with zero runtime overhead, tree-shakes unused messages, and integrates seamlessly with SvelteKit's routing.

> 选择 Paraglide JS 是因为它专为 SvelteKit 设计，编译时生成类型安全的翻译函数，零运行时开销。

### Why Paraglide over alternatives?

| Feature | Paraglide JS | svelte-i18n | i18next |
|---------|-------------|-------------|---------|
| Bundle size | ~0 KB (compiled away) | ~5 KB | ~40 KB |
| Type safety | Full (generated functions) | Partial | Plugin needed |
| SvelteKit SSR | Native support | Manual setup | Adapter needed |
| Tree-shaking | Per-message level | No | No |
| URL-based routing | Built-in plugin | Manual | Manual |

## Supported Languages

| Language | Code | Default | Notes |
|----------|------|---------|-------|
| Chinese (Simplified) 中文 | `zh` | Yes | Primary audience is Chinese community / 主要受众为华人社区 |
| English | `en` | No | Switchable via language toggle / 可通过语言切换器切换 |

Chinese is the default because the primary audience is the Saskatoon Chinese community, but English is fully supported and accessible via the language switcher at any time.

## Installation

```bash
# Install Paraglide SvelteKit plugin
pnpm add @inlang/paraglide-sveltekit

# Initialize inlang project (creates project.inlang/ directory)
npx @inlang/paraglide-js init
```

### Inlang Project Settings

```json
// project.inlang/settings.json
{
  "$schema": "https://inlang.com/schema/project-settings",
  "sourceLanguageTag": "zh",
  "languageTags": ["zh", "en"],
  "modules": [
    "https://cdn.jsdelivr.net/npm/@inlang/message-lint-rule-empty-pattern@latest/dist/index.js",
    "https://cdn.jsdelivr.net/npm/@inlang/message-lint-rule-missing-translation@latest/dist/index.js",
    "https://cdn.jsdelivr.net/npm/@inlang/plugin-message-format@latest/dist/index.js",
    "https://cdn.jsdelivr.net/npm/@inlang/plugin-m-function-matcher@latest/dist/index.js"
  ],
  "plugin.inlang.messageFormat": {
    "pathPattern": "./messages/{languageTag}.json"
  }
}
```

**Modules explained:**
- `empty-pattern` — Lint rule: warns when a translation key has an empty value
- `missing-translation` — Lint rule: warns when a key exists in one language but not the other
- `plugin-message-format` — Reads/writes the `messages/*.json` files
- `plugin-m-function-matcher` — IDE support for `m.*()` function references

### SvelteKit Plugin Integration

```js
// vite.config.ts
import { paraglide } from '@inlang/paraglide-sveltekit/vite';
import { sveltekit } from '@sveltejs/kit/vite';
import { defineConfig } from 'vite';

export default defineConfig({
  plugins: [
    paraglide({
      project: './project.inlang',
      outdir: './src/lib/paraglide'
    }),
    sveltekit()
  ]
});
```

// src/hooks.server.ts
import { i18n } from '$lib/i18n';
import type { Handle } from '@sveltejs/kit';

// src/lib/i18n.ts
import { createI18n } from '@inlang/paraglide-sveltekit';

## Translation Files / 翻译文件

Translation messages are stored as flat JSON files under `messages/`. Each key uses a **namespaced prefix** to organize by feature area.

> 翻译消息以扁平 JSON 文件存储在 `messages/` 目录下，每个 key 使用命名空间前缀按功能区分。

### Key Naming Convention

| Prefix | Area | Example |
|--------|------|---------|
| `site_` | Global site info | `site_title`, `site_subtitle` |
| `nav_` | Navigation links | `nav_home`, `nav_register` |
| `hero_` | Homepage hero section | `hero_date`, `hero_cta` |
| `category_` | Tournament categories | `category_100`, `category_80` |
| `reg_` | Registration form | `reg_title`, `reg_submit` |
| `teams_` | Teams listing page | `teams_title`, `teams_empty` |
| `rules_` | Tournament rules | `rules_title`, `rules_format` |
| `sponsors_` / `sponsor_` | Sponsors page | `sponsors_title`, `sponsor_gold` |
| `footer_` | Footer content | `footer_organizer` |
| `common_` | Shared UI strings | `common_loading`, `common_cancel` |

## Usage in Components / 组件中使用翻译

Paraglide generates typed `m.*()` functions from the message keys. Import them from the generated `$lib/paraglide/messages` module.

> Paraglide 从翻译 key 生成类型安全的 `m.*()` 函数，从 `$lib/paraglide/messages` 导入使用。

### Basic Usage

```svelte
<script lang="ts">
  import * as m from '$lib/paraglide/messages';
</script>

<h1>{m.site_title()}</h1>
<p>{m.site_subtitle()}</p>
<a href="/register">{m.nav_register()}</a>
```

### With Parameters (ICU MessageFormat)

For messages containing placeholders like `{count}`, pass an object:

```svelte
<script lang="ts">
  import * as m from '$lib/paraglide/messages';

  let spotsLeft = 24;
</script>

<p>{m.hero_spots_remaining({ count: spotsLeft })}</p>
<!-- zh: "剩余 24 个名额" -->
<!-- en: "24 spots remaining" -->
```

### In TypeScript Files

```ts
// src/lib/utils.ts
import * as m from '$lib/paraglide/messages';

export function getValidationError(type: string): string {
  switch (type) {
    case 'age':
      return m.reg_age_error();
    case 'sum':
      return m.reg_sum_error();
    case 'duplicate':
      return m.reg_duplicate_error();
    case 'full':
      return m.reg_full_error();
    default:
      return m.common_error();
  }
}
```

### Translated Links with `i18n.route()`

When using `<a>` tags, wrap `href` with `i18n.route()` to ensure the URL includes the correct language prefix:

```svelte
<script lang="ts">
  import { i18n } from '$lib/i18n';
  import * as m from '$lib/paraglide/messages';
</script>

<a href={i18n.route('/register')}>{m.nav_register()}</a>
<!-- zh: href="/register" (default, no prefix) -->
<!-- en: href="/en/register" -->
```

### Reactive Language Display

Use `languageTag()` from the Paraglide runtime to get the current language:

```svelte
<script lang="ts">
  import { languageTag } from '$lib/paraglide/runtime';
</script>

<!-- Show the appropriate name based on current language -->
{#if languageTag() === 'zh'}
  <span class="font-chinese">{player.name_zh}</span>
{:else}
  <span>{player.name_en}</span>
{/if}
```

## Language Switcher Component / 语言切换组件

```svelte
<!-- src/lib/components/LanguageSwitcher.svelte -->
<script lang="ts">
  import { languageTag, setLanguageTag, availableLanguageTags } from '$lib/paraglide/runtime';

  const labels: Record<string, string> = {
    zh: '中文',
    en: 'EN'
  };
</script>

<div class="flex gap-2">
  {#each availableLanguageTags as lang}
    <button
      class="px-3 py-1 rounded text-sm font-medium transition-colors
        {languageTag() === lang
          ? 'bg-primary text-white'
          : 'bg-gray-200 text-gray-700 hover:bg-gray-300'}"
      on:click={() => setLanguageTag(lang)}
    >
      {labels[lang]}
    </button>
  {/each}
</div>
```

The switcher will be placed in the `Header.svelte` component, replacing the current non-functional `中/EN` button.

> 语言切换器将放置在 `Header.svelte` 中，替换当前无功能的 `中/EN` 按钮。

## URL Strategy / URL 策略

We use a **path prefix** strategy, handled automatically by the Paraglide SvelteKit plugin:

> 使用路径前缀方案，由 Paraglide SvelteKit 插件自动处理。

| URL | Language | Notes |
|-----|----------|-------|
| `https://example.com/register` | Chinese (default) | No prefix for default language |
| `https://example.com/en/register` | English | `/en` prefix for English |
| `https://example.com/zh/register` | Chinese | Explicit `/zh` also works |

The plugin automatically:
- Adds `<link rel="alternate" hreflang="...">` tags for SEO
- Handles redirects between language versions
- Preserves the selected language across navigation
- Sets `lang` attribute on the `<html>` element

### Route Configuration

```ts
// src/lib/i18n.ts
import { createI18n } from '@inlang/paraglide-sveltekit';
import * as runtime from '$lib/paraglide/runtime';

export const i18n = createI18n(runtime, {
  // Paths that should NOT be prefixed (API routes, etc.)
  exclude: ['/api/*'],
  // Default language behavior: no prefix for default language
  prefixDefaultLanguage: 'never'
});
```

## Database Bilingual Fields / 数据库双语字段

Database fields that need bilingual content use `_zh` / `_en` suffixes:

> 数据库中需要双语的字段采用 `_zh` / `_en` 后缀。

| Table | Chinese Field | English Field |
|-------|--------------|---------------|
| `categories` | `name_zh` | `name_en` |
| `sponsors` | `name_zh` | `name_en` |
| `players` | `name_zh` | `name_en` |
| `teams` | `team_name_zh` | `team_name_en` |

### Rendering Based on Language

```svelte
<script lang="ts">
  import { languageTag } from '$lib/paraglide/runtime';

  // Helper to pick the right field
  function localized(zh: string, en: string): string {
    return languageTag() === 'zh' ? zh : en;
  }
</script>

<!-- Usage in templates -->
<h3>{localized(sponsor.name_zh, sponsor.name_en)}</h3>
<p>{localized(category.name_zh, category.name_en)}</p>
```

A reusable helper can also be created:

```ts
// src/lib/utils.ts
import { languageTag } from '$lib/paraglide/runtime';

/**
 * Returns the appropriate bilingual field value based on the current language.
 * Falls back to the other language if the preferred one is empty.
 */
export function localized(zh: string | null, en: string | null): string {
  if (languageTag() === 'zh') {
    return zh || en || '';
  }
  return en || zh || '';
}
```

## SEO Considerations / SEO 考量

### Hreflang Tags

The Paraglide SvelteKit plugin automatically injects `hreflang` alternate links:

```html
<!-- Auto-generated in <head> -->
<link rel="alternate" hreflang="zh" href="https://example.com/register" />
<link rel="alternate" hreflang="en" href="https://example.com/en/register" />
<link rel="alternate" hreflang="x-default" href="https://example.com/register" />
```

### HTML Lang Attribute

The `<html lang="...">` attribute is set automatically based on the current route:

```html
<!-- Chinese page -->
<html lang="zh">

<!-- English page -->
<html lang="en">
```

### OpenGraph / Meta Tags

Use Paraglide messages in `+page.ts` or `+page.server.ts` for translated meta:

```ts
// src/routes/+page.ts
import * as m from '$lib/paraglide/messages';

export function load() {
  return {
    title: m.site_title(),
    description: m.site_subtitle()
  };
}
```

```svelte
<!-- src/routes/+layout.svelte -->
<svelte:head>
  <title>{$page.data.title ?? m.site_title()}</title>
  <meta name="description" content={$page.data.description ?? m.site_subtitle()} />
  <meta property="og:locale" content={languageTag() === 'zh' ? 'zh_CN' : 'en_CA'} />
</svelte:head>
```

## Font Strategy / 字体策略

Chinese and English text use different font stacks, defined in CSS custom properties:

```css
/* src/routes/layout.css */
--font-heading: 'Bebas Neue', 'Noto Sans SC', sans-serif;
--font-body: 'Source Sans 3', 'Noto Sans SC', sans-serif;
--font-chinese: 'Noto Sans SC', sans-serif;
```

Use the `font-chinese` class for Chinese-only text (e.g., names) to ensure proper rendering. The body and heading fonts include `Noto Sans SC` as a fallback for any Chinese characters within mixed-language content.

## Adding New Translation Keys / 添加新翻译 Key

When adding a new translatable string:

1. **Add the key to both `messages/zh.json` and `messages/en.json`**
2. **Use the appropriate prefix** (see naming convention table above)
3. **Run the dev server** — Paraglide auto-generates the `m.*()` function
4. **Import and use** `m.your_new_key()` in the component
5. **Lint check** — the `missing-translation` rule will catch keys that exist in only one language

```bash
# Check for missing translations
npx @inlang/cli lint
```
